Bases de datos relacionales

Los datos en las empresas suelen estar en estos sistemas gestores de bases de datos, que requieren unos datos de conexión y a partir de ahí, los datos se pueden extraer con expresiones SQL. En el proyecto se van a crear bases de datos “locales” para practicar su creación y modificación, y se utilizará una base de datos remota para leer datos en tiempo real desde R (sin necesidad de conocer SQL).

Vamos a seguir los ejemplos y explicaciones del Capítulo 21 de R for Data Science 2nd ed., aplicándolos a nuestros conjuntos de datos de vino.

Esquema relacional: múltiples tablas

En un sistema real los datos raramente están en una sola tabla. Lo habitual es tener un esquema normalizado: tablas pequeñas de referencia enlazadas mediante claves foráneas a la tabla principal.

Vamos a descomponer los datos de catas en tres tablas:

Tabla Contenido Clave primaria
tasting Una fila por reseña id
tasters Un catador por fila taster_name
produccion Producción mundial por país y año pais + anio
dos_espana Denominaciones de Origen españolas do 
# Leer los ficheros limpios producidos por el ETL
winetasting    <- readRDS("datos/clean/tasting.rds")
produccion_vino <- readRDS("datos/clean/produccion_vino.rds")
dos_espana     <- readRDS("datos/clean/dos_espana.rds")

# Con parquet (alternativa, requiere {arrow}):
# library(arrow)
# winetasting     <- read_parquet("datos/clean/tasting.parquet")
# produccion_vino <- read_parquet("datos/clean/produccion_vino.parquet")
# dos_espana      <- read_parquet("datos/clean/dos_espana.parquet")
# Tabla principal de reseñas
dbWriteTable(con, "tasting", as.data.frame(winetasting), overwrite = TRUE)

# Tabla de catadores únicos
tasters <- winetasting |>
  distinct(taster_name, taster_twitter_handle) |>
  drop_na(taster_name)

dbWriteTable(con, "tasters", as.data.frame(tasters), overwrite = TRUE)

# Tabla de producción mundial (datos reales de Our World in Data / FAO)
dbWriteTable(con, "produccion", as.data.frame(produccion_vino), overwrite = TRUE)

# Tabla de DOs españolas
dbWriteTable(con, "dos_espana", as.data.frame(dos_espana), overwrite = TRUE)

Consultas SQL directas

Se pueden ejecutar expresiones SQL directamente. En este ejemplo dbGetQuery() envía la consulta y trae los datos a R en un solo paso.

# Filtro simple
consulta <- "SELECT * FROM tasting WHERE country = 'Spain'"
dbGetQuery(con, consulta)

JOIN entre tablas

El poder real de las bases de datos relacionales está en los JOINs: combinar filas de distintas tablas usando una clave común.

# INNER JOIN: reseñas con información del catador
consulta_join <- "
  SELECT t.id, t.title, t.points, t.price, t.country,
         s.taster_name, s.taster_twitter_handle
  FROM tasting AS t
  INNER JOIN tasters AS s ON t.taster_name = s.taster_name
  WHERE t.country = 'Spain'
  ORDER BY t.points DESC
  LIMIT 20
"
dbGetQuery(con, consulta_join)
# LEFT JOIN: reseñas ibéricas enriquecidas con producción del país en 2020
consulta_prod <- "
  SELECT t.id, t.country, t.points, t.price, t.variety,
         p.produccion_toneladas
  FROM tasting AS t
  LEFT JOIN produccion AS p
    ON t.country = p.pais AND p.anio = 2020
  WHERE t.country IN ('Spain', 'Portugal')
  ORDER BY p.produccion_toneladas DESC
"
dbGetQuery(con, consulta_prod)
Tip

Un INNER JOIN devuelve solo filas con coincidencia en ambas tablas. Un LEFT JOIN devuelve todas las filas de la tabla izquierda aunque no haya coincidencia en la derecha (los valores sin coincidencia aparecen como NA).

Trabajar en la base de datos desde R con {dbplyr}

En vez de usar las funciones de {DBI}, podemos usar las de {dbplyr} y {dplyr}. La función tbl() crea un vínculo con la tabla de la base de datos, pero no la “trae” a la memoria de R. Así podemos explorar los datos antes de traer lo que nos interese con collect().

winetbl   <- tbl(con, "tasting")
tasterbl  <- tbl(con, "tasters")
prodbl    <- tbl(con, "produccion")

Filtros y ordenación

winetbl |>
  filter(country == "Spain") |>
  arrange(desc(points))

Ver la consulta SQL equivalente que genera {dbplyr}:

winetbl |>
  filter(country == "Spain") |>
  arrange(desc(points)) |>
  show_query()

JOIN con {dplyr} / {dbplyr}

La sintaxis es idéntica a un left_join() normal de dplyr, pero se ejecuta en la base de datos:

# Unir reseñas con datos del catador.
# Eliminamos taster_twitter_handle de tasterbl antes del join para evitar
# columnas duplicadas (ambas tablas la tienen). La columna del resultado
# viene de winetbl.
vinos_con_catador <- winetbl |>
  left_join(tasterbl |> select(-taster_twitter_handle), by = "taster_name") |>
  filter(country == "Spain") |>
  select(title, points, price, taster_name, taster_twitter_handle)

vinos_con_catador |> show_query()
vinos_con_catador |> collect()
# Reseñas ibéricas enriquecidas con producción 2020
vinos_produccion <- winetbl |>
  filter(country %in% c("Spain", "Portugal")) |>
  left_join(
    prodbl |> filter(anio == 2020),
    by = c("country" = "pais")
  ) |>
  select(country, title, points, price, produccion_toneladas) |>
  arrange(desc(produccion_toneladas))

vinos_produccion |> collect()

Agrupación y resumen

winetbl |>
  filter(country %in% c("Spain", "Portugal")) |>
  left_join(prodbl |> filter(anio == 2020), by = c("country" = "pais")) |>
  group_by(country, produccion_toneladas) |>
  summarise(
    n_resenas    = n(),
    puntos_medio = mean(points, na.rm = TRUE),
    precio_medio = mean(price,  na.rm = TRUE),
    .groups      = "drop"
  ) |>
  collect()

Para traer el resultado a R, función collect() (hasta entonces los datos solo están en la base de datos).

vinos_es <- winetbl |>
  filter(country == "Spain") |>
  collect()

PrecauciónEjercicio

Usando la tabla dos_espana, obtén la puntuación media por comunidad autónoma para los vinos españoles. Ordena el resultado de mayor a menor puntuación.

TipSolución 
dostbl <- tbl(con, "dos_espana")

winetbl |>
  filter(country == "Spain") |>
  left_join(dostbl, by = c("region_1" = "do")) |>
  filter(!is.na(comunidad)) |>
  group_by(comunidad) |>
  summarise(
    n_resenas    = n(),
    puntos_medio = mean(points, na.rm = TRUE),
    precio_medio = mean(price,  na.rm = TRUE),
    .groups      = "drop"
  ) |>
  arrange(desc(puntos_medio)) |>
  collect()

Recomendaciones:
